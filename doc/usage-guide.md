# AI Image Studio - 项目使用说明

## 项目简介

AI Image Studio 是一个基于 Web 的 AI 图片生成平台，用户通过浏览器输入提示词即可生成图片。系统包含用户登录、生图队列管理、图片预览与历史记录等功能。

---

## 环境要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Nginx | 任意稳定版 | 已安装在 `C:\nginx` |
| Node.js | >= 18 | 已安装，当前 v22.14.0 |
| Stable Diffusion WebUI | 可选 | 图片生成后端，端口 7860 |

---

## 启动步骤

整个系统需要启动 **3 个服务**，按以下顺序操作：

### 第一步：启动 Nginx

```bash
# 方式一：双击运行
C:\nginx\nginx.exe

# 方式二：命令行
cd C:\nginx
nginx.exe
```

验证：打开浏览器访问 `http://localhost`，应看到登录页面。

如需重启 Nginx：
```bash
C:\nginx\nginx.exe -p C:\nginx -s reload
```

如需停止 Nginx：
```bash
C:\nginx\nginx.exe -p C:\nginx -s quit
```

### 第二步：启动 Node.js 后端

```bash
# 方式一：双击 bat 文件
C:\NginxData\server\start.bat

# 方式二：命令行
cd C:\NginxData\server
node server.js
```

启动成功后会看到：
```
AI Image Server running on http://127.0.0.1:8000
```

> 此窗口需保持打开，关闭后后端停止。可用 `Ctrl+C` 终止。

### 第三步：启动图片生成后端（可选）

如果你部署了 Stable Diffusion WebUI，启动它并确保监听在 `7860` 端口：

```bash
# Stable Diffusion WebUI 启动示例（根据你的实际安装路径调整）
cd C:\sd-webui
webui-user.bat
```

> 如果没有启动图片生成后端，生图任务会报错，但不影响登录和界面操作。

## 使用流程

### 1. 访问页面

浏览器打开：

```
http://localhost        # 本机访问
http://你的IP           # 局域网内其他设备访问
```

### 2. 登录

进入页面后看到登录界面：

1. 在输入框中输入用户名（支持中文、英文、数字及其组合）
2. 点击 **"进入工作室"** 按钮
3. 用户名会与你的 IP 地址绑定

```
注意事项：
- 用户名不能为空
- 同一 IP 只能使用同一个用户名
- 同一用户名只能在一台设备上使用
- 登录状态会保存在浏览器中，刷新页面无需重新登录
```

### 3. 生成图片

登录后进入生图界面，左侧为操作面板，右侧为预览区域：

**操作步骤：**

1. **输入提示词** — 在文本框中用英文描述你想生成的图片
   - 示例：`a cute cat sitting on a windowsill, warm sunlight, watercolor style`
   - 支持 `Ctrl + Enter` 快捷提交

2. **选择图片尺寸** — 点击选择：
   - `512 x 512` — 标准尺寸，生成快
   - `1024 x 1024` — 高清尺寸
   - `1024 x 768` — 横屏比例
   - `768 x 1024` — 竖屏比例

3. **选择生成数量** — 可选 1、2 或 4 张

4. **点击 "开始生成"** — 任务进入队列

### 4. 查看队列状态

提交后可以在左侧下方 **"我的生图队列"** 面板中看到：

- 当前任务在全局队列中的 **排名**
- 任务状态：
  - **排队中**（黄色）— 等待处理
  - **生成中**（蓝色，闪烁）— 正在生成
  - **已完成**（绿色）— 可以查看图片
  - **失败**（红色）— 生成出错

导航栏右上角显示 **"队列 X/50"**，表示当前全局队列占用情况。

### 5. 查看图片

- 生成完成后，图片自动显示在右侧 **预览区域**
- 鼠标悬停在图片上可以看到 **下载** 和 **新窗口打开** 按钮
- 下方 **历史记录** 区域展示该用户所有已生成的图片缩略图
- 点击缩略图可在预览区查看大图

---

## 队列限制规则

| 限制类型 | 数值 | 说明 |
|----------|------|------|
| 单用户最大排队 | **3** | 每个用户最多同时 3 个任务（排队 + 生成中） |
| 全局最大排队 | **50** | 所有用户合计最多 50 个活跃任务 |

超出限制时系统会提示具体原因，需等待现有任务完成后才能继续提交。

## 常见问题

### Q: 页面打不开？

1. 检查 Nginx 是否启动：任务管理器中查找 `nginx.exe` 进程
2. 检查防火墙是否放行 80 端口
3. 检查 Nginx 配置：`C:\nginx\nginx.exe -p C:\nginx -t`

### Q: 登录失败 / 提示 IP 已绑定？

- 同一 IP 只能绑定一个用户名，系统会提示已绑定的用户名
- 换一个用户名或使用提示的用户名登录
- 如需解绑，重启 Node.js 后端（内存中的绑定关系会清除）

### Q: 点击生成后报错？

- 确认 Node.js 后端已启动（命令行窗口显示 `AI Image Server running...`）
- 确认图片生成后端（SD WebUI）已启动在 `7860` 端口
- 检查 Node.js 后端的命令行窗口是否有错误日志

### Q: 图片生成很慢？

- 取决于 Stable Diffusion WebUI 的性能和显卡
- 高分辨率（1024x1024）比低分辨率（512x512）慢
- 批量生成（2或4张）比单张慢
- 可在 `server.js` 中调整 `steps` 参数（默认20，越低越快但质量越差）

### Q: 服务重启后需要重新登录吗？

- 是的。Token 和 IP 绑定关系存储在内存中，重启后失效
- 用户文件夹和已生成的图片不受影响

### Q: 如何修改队列限制？

编辑 `C:\NginxData\server\server.js` 文件顶部：

```javascript
const MAX_USER_QUEUE = 3;    // 单用户最大并发，改为需要的值
const MAX_GLOBAL_QUEUE = 50; // 全局最大并发，改为需要的值
```

修改后需重启 Node.js 后端。

### Q: 如何换用其他图片生成模型？

参考 `C:\NginxData\doc\backend-queue-api.md` 中「接入第三方图片生成后端」章节，修改 `server.js` 中的 `processQueueItem` 函数即可对接 ComfyUI、DALL-E 等。
